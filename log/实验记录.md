### 顶层设计

本项目设计一个 **ASR Harness**：在冻结预训练 ASR 模型的基础上，通过模型外部流程提升识别效果。

Harness 主要包含四个模块：音频规范化、上下文/热词提示、文本纠错、候选重排序。实验采用同一预训练模型，对比以下两种流程：

* **Baseline**：音频 → 预训练模型 → 文本
* **Harness**：音频 → Harness 增强 → 预训练模型 → Harness 后处理 → 文本

最终通过 CER 对比验证 Harness 是否带来有效提升。该方案避免大规模训练，重点体现系统设计与推理阶段优化。


### 模型

### 2026-06-11 Harness v1：音频多视图 + 字符 n-gram 重排序

设计原则：

1. 不训练或微调 ASR 模型。
2. 不使用 `test` 标注构建纠错表。
3. 不做面向 `test` 的规则替换。
4. 候选只来自同一 ASR 后端对同一音频的不同输入视图。
5. 重排序只使用 `train` 文本构建的字符 n-gram LM。

Harness v1 流程：

```text
音频
  -> 生成多个音频视图
  -> 同一 ASR 模型分别识别
  -> 得到候选文本集合
  -> 用 full/train 构建的字符 n-gram LM 给候选打分
  -> 加入 score margin，置信度不足时回退原始候选
  -> 输出最终文本并计算 CER
```

默认音频视图：

| 视图 | 说明 |
| --- | --- |
| `orig` | 原始音频 |
| `peak_norm` | 峰值归一化 |
| `trim_peak_norm` | 静音裁剪 + 峰值归一化 |
| `preemph` | 预加重 |
| `speed_0_95` | 轻微降速 |

曾尝试但移出默认配置的视图：

| 视图 | 原因 |
| --- | --- |
| `highpass` | 对部分样本引入过强失真 |
| `speed_1_05` | 加速视图稳定性差于轻微降速 |

新增文件：

| 文件 | 功能 |
| --- | --- |
| `harness/src/audio/variants.py` | 生成音频多视图 |
| `harness/src/lm/char_ngram.py` | 从 manifest 构建带回退的字符 n-gram LM |
| `harness/src/pipeline/harness.py` | Harness v1 推理与评测 |
| `harness/scripts/evaluate_harness.py` | Harness v1 命令行入口 |
| `harness/scripts/rescore_harness_predictions.py` | 对已保存候选离线重排序 |
| `harness/scripts/summarize_harness.py` | 汇总 Harness 结果 |

重要实现修正：

1. 初版 4-gram 太稀疏，很多候选分数相同。
2. 已改为带回退的字符 n-gram：4-gram 未命中则退到 3/2/1-gram。
3. 增加 `--score-margin`，避免 LM 分数微弱领先时误切换候选。

| `candidate_consensus` | 在多个模型候选中按模型权重和与其它候选的一致性选一个候选 |
| `weighted_vote` | 类 ROVER 字符级加权投票，可能生成新文本 |
| `cascade_fast_exact` | 快模型完全一致时直接采用快模型；否则回退 `qwen3-asr-1.7b` |
| `oracle` | 仅用于分析上限，使用参考答案选择最优候选，不能作为真实 Harness |

正式权重：

```text

qwen3-asr-0.6b = 2.0
paraformer-baseline = 1.0
paraformer-harness-v1-margin015 = 1.0
```

4. 当前最合理的 Harness v3 不是“投票覆盖强模型”，而是双模式：
   - 精度优先：直接采用 `qwen3-asr-1.7b`，同时保留其它模型候选用于审计。
   - 成本优先：使用 `cascade_fast_exact`，快模型完全一致时不调用 1.7B，否则调用 1.7B；代表集上只需 47% 的 1.7B 调用，CER 为 0.027133。
5. 后续若要真正超过 1.7B，需要引入模型置信度、token/logprob、n-best 或专名/实体校正信号；仅靠 top1 文本投票不够。

### 2026-06-12 最终报告重构与补充级联实验

本次根据报告写作目标，将结论从“多个小模型媲美大模型”调整为更稳健的工程命题：

> 多个较快模型辅助大模型决策，让大模型只处理快模型分歧样本，从而在维持强模型级 CER 的同时降低系统平均运行时间。

本次不引入 Kaldi/GMM-HMM 训练流程。原因是标准 GMM-HMM 需要额外 recipe、词典、特征、解码图和 Linux/WSL 环境，工程改动过大。补充实验选择已下载且可运行的 `vosk-full` 作为 Kaldi 生态/传统离线识别器代表。

补跑 `representative100_eval/test` 快模型结果：

| 模型 | 类型 | 样本数 | CER | 编辑距离 | 平均推理耗时 |
| --- | --- | ---: | ---: | ---: | ---: |
| `qwen3-asr-1.7b` | 强模型 | 100 | 0.026447 | 85 | 13.0793s |
| `qwen3-asr-0.6b` | 中等规模模型 | 100 | 0.038892 | 125 | 1.6239s |
| `paraformer-zh` | 中文快模型 | 100 | 0.040448 | 130 | 0.1002s |
| `sensevoice-small` | 快模型 | 100 | 0.052271 | 168 | 0.0763s |
| `sherpa-onnx-paraformer-zh-small` | CPU/ONNX 快模型 | 100 | 0.071873 | 231 | 0.0976s |
| `vosk-full` | Kaldi 生态离线识别器 | 100 | 0.072495 | 233 | 1.5404s |

最终级联 Harness 设计：

```text
先运行 qwen3-asr-0.6b 与 paraformer-zh
若两者规范化输出完全一致，则直接采用 qwen3-asr-0.6b
若两者输出分歧，则调用 qwen3-asr-1.7b
```

最终级联命令：

```powershell
.\.venv\Scripts\python.exe harness\scripts\evaluate_ensemble_harness.py --manifest data\manifests\representative100_eval\test.jsonl --prediction qwen3-asr-1.7b=3.0=outputs\representative100_eval\qwen_asr\qwen3-asr-1.7b\test_predictions.jsonl --prediction qwen3-asr-0.6b=2.0=outputs\representative100_eval\qwen_asr\qwen3-asr-0.6b\test_predictions.jsonl --prediction paraformer-zh=1.0=outputs\representative100_eval\baseline\paraformer-zh\test_predictions.jsonl --anchor qwen3-asr-1.7b --strategy cascade_fast_exact --output-root outputs --harness-name harness_final_eval --output-name cascade_qwen06_para_exact
```

最终级联结果：

| 方法 | CER | 编辑距离 | 1.7B 调用率 | 平均耗时估计 |
| --- | ---: | ---: | ---: | ---: |
| `qwen3-asr-1.7b` | 0.026447 | 85 | 100% | 13.0793s |
| `cascade_qwen06_para_exact` | 0.026136 | 84 | 55% | 8.9177s |

```text
T = T_qwen0.6 + T_paraformer + 0.55 * T_qwen1.7
  = 1.6239 + 0.1002 + 0.55 * 13.0793
  = 8.9177s
```
