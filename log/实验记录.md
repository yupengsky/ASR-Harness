### 顶层设计

本项目设计一个 **ASR Harness**：在冻结预训练 ASR 模型的基础上，通过模型外部流程提升识别效果。

Harness 主要包含四个模块：音频规范化、上下文/热词提示、文本纠错、候选重排序。实验采用同一预训练模型，对比以下两种流程：

* **Baseline**：音频 → 预训练模型 → 文本
* **Harness**：音频 → Harness 增强 → 预训练模型 → Harness 后处理 → 文本

最终通过 CER 对比验证 Harness 是否带来有效提升。该方案避免大规模训练，重点体现系统设计与推理阶段优化。


### 模型

本项目不从零训练模型，而是选取多个开源预训练 ASR 模型作为识别后端，用于验证 Harness 在不同模型上的适用性。模型选择遵循三个原则：本地可运行、中文识别能力较强、推理成本可控。

已下载并纳入候选的模型如下：

| 模型                              | 类型          | 选择原因                   | 项目用途          |
| ------------------------------- | ----------- | ---------------------- | ------------- |
| Paraformer-zh                   | 中文端到端 ASR   | 中文识别能力强，推理速度较快         | 主力模型          |
| SenseVoiceSmall                 | 多语种语音理解/ASR | 支持中文、中英混合与多种语音信息       | 主力模型/对比模型     |
| Whisper tiny                    | 多语种 ASR     | 模型极小，运行成本低             | 轻量 baseline   |
| Whisper base                    | 多语种 ASR     | 精度与成本较平衡               | 通用 baseline   |
| Whisper small                   | 多语种 ASR     | 精度高于 tiny/base，但仍可本地尝试 | 高精度对比         |
| Vosk small/full 中文模型            | 离线 ASR      | CPU 可运行，部署简单           | 传统轻量 baseline |
| Moonshine Tiny zh               | 轻量中文 ASR    | 模型小，适合低显存环境            | 轻量对比          |

### 模型测试

模型测试分为两个阶段：**可运行性测试**和**识别效果测试**。

首先进行可运行性测试，确认每个预训练模型能否在本地环境中正常加载并完成单条音频推理。测试内容包括模型加载是否成功、是否发生显存不足、单条音频推理耗时、输出文本是否正常。无法稳定运行的模型不进入后续 CER 对比。

随后进行识别效果测试。选取 THCHS30 测试集中的若干音频样本，分别输入各个可运行模型，记录模型原始输出文本，并与标准标注文本计算 CER。该阶段用于筛选主力模型，并确定后续 Harness 增强实验的基础模型。

1. `paraformer-zh` 是最适合作为 Harness 主力实验对象的后端。
