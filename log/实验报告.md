# 中文自动语音识别多模型分级识别系统项目报告

## 摘要

本项目面向中文自动语音识别任务，输入一段语音，输出对应中文文本。数据集采用 THCHS30，主要评价指标为字错误率 CER。项目保持本地预训练 ASR 模型参数冻结，改进集中在推理阶段的系统设计、统一评测和多模型决策。

最终系统采用“多模型分级识别系统”：先运行 Qwen3-ASR-0.6B 与 Paraformer-zh 两个较快模型，对两者输出进行同一中文文本规范化；若规范化文本完全一致，直接采用 Qwen3-ASR-0.6B 输出；若两者存在分歧，再调用 Qwen3-ASR-1.7B，并采用 1.7B 输出。该设计用快模型一致性作为稳定性信号，通过一致性门控控制强模型调用率，在低 CER 与低平均运行时间之间取得折中。

在THCHS30中选取的评估子集上，Qwen3-ASR-1.7B 单模型 CER 为 0.026447。最终分级系统 CER 为 0.026136，Qwen3-ASR-1.7B 调用率为 55%，多小模型分级系统相比于单个较大模型的CER降幅很小。但是相比每条都调用 1.7B，运行时间降低约 31.82%。有明显的收益。
## 1. 任务与数据

项目任务是构建中文自动语音识别系统：

```text
输入：音频文件
输出：中文识别文本
```

数据集使用 THCHS30。工程通过 `harness/scripts/prepare_manifests.py` 从原始目录生成统一 manifest，并通过 `harness/scripts/validate_manifests.py` 校验音频路径、文本字段和重复样本。manifest 每条样本包含 `utt_id`、`split`、`wav_path`、`text` 四个字段，结构定义见 `harness/src/data/manifest.py`。

`data/manifests/full/stats.json` 中记录的完整集统计如下。

| split | 样本数 | 最短参考文本 | 最长参考文本 | 平均参考文本长度 |
| --- | ---: | ---: | ---: | ---: |
| full/train | 10000 | 20 | 47 | 32.63 |
| full/dev | 893 | 20 | 47 | 32.67 |
| full/test | 2495 | 19 | 48 | 32.52 |

其中 full/test 共 2495 条，平均参考文本长度约 32.52 个中文字符。

由于 Qwen3-ASR-1.7B 推理耗时较高，项目另外构建了 `representative100_eval/test` 作为受限评估子集，用于强模型与多模型决策实验。该子集包含 100 条样本，来源于 full/test，并排除了前期开发用的 `representative100/test`。该子集用于控制高耗时实验成本，不作为完整 test 集替代品。

## 2. 评价指标

主要指标为 CER，越低越好。工程实现见 `harness/src/metrics/cer.py`。

文本规范化口径为：

1. 删除空白字符。
2. 删除标点、数字、英文和其他非中文字符。
3. 仅保留 Unicode 中文字符范围 `\u4e00-\u9fff`。

编辑距离 ED 定义为：将识别文本变成参考文本所需的最少字符替换、删除、插入次数。

CER 计算公式为：

```text
CER = ED / 参考文本字符数
```

当参考文本为空时，工程按特殊情况处理；THCHS30 当前 manifest 中参考文本均非空。

## 3. 工程实现

项目实现了一套统一评测框架，使多个本地预训练 ASR 模型共享同一数据读取、推理输出、指标计算和结果保存格式。

统一内容包括：

| 模块 | 作用 | 主要文件 |
| --- | --- | --- |
| manifest 数据格式 | 统一样本字段与路径校验 | `harness/src/data/manifest.py` |
| 模型注册表 | 管理本地模型路径、后端类型和优先级 | `harness/configs/models.json` |
| ASR 后端接口 | 统一 `transcribe(wav_path)` 与 `ASRResult` | `harness/src/models/base.py` |
| 后端工厂 | 按模型名创建 FunASR、Vosk、Sherpa-ONNX、Transformers 后端 | `harness/src/models/factory.py` |
| 中文规范化与 CER | 统一文本规范化、编辑距离和 CER 统计 | `harness/src/metrics/cer.py` |
| baseline 评测 | 批量运行模型、保存 JSONL 与 summary JSON | `harness/src/pipeline/baseline.py` |
| 多视图候选实验 | 生成音频视图并用字符 n-gram LM 重排序 | `harness/src/pipeline/harness.py` |
| 多模型候选融合 | 离线候选池、投票、共识和理论上限分析 | `harness/src/pipeline/fusion.py` |
| 分级决策评测 | 级联、加权投票、oracle 等策略评测 | `harness/scripts/evaluate_ensemble_harness.py` |
| 单条音频入口 | 实际使用场景下输出最终文本与各模型信息 | `harness/scripts/transcribe_one.py` |

批量评测会输出两类文件：

| 文件 | 内容 |
| --- | --- |
| `*_predictions.jsonl` | 每条音频的参考文本、识别文本、规范化文本、ED、ref_chars、CER、耗时 |
| `*_summary.json` | 样本数、总编辑距离、总参考字符数、整体 CER、模型加载耗时、总音频时长、总推理耗时 |

单条音频入口 `harness/scripts/transcribe_one.py` 支持实际使用场景。只输入音频时，输出最终识别文本和各模型文本；同时输入参考文本时，额外输出 ED、ref_chars、CER。输出 JSON 包含 `final_text`、`final_norm`、`selected_source`、`qwen17_called`、`decision`、`models`，以及可选 `metrics`。

示例命令：

```powershell
.\.venv\Scripts\python.exe harness\scripts\transcribe_one.py --audio "data\mydata\xxx.m4a"
```

## 4. 模型与后端

工程统一接入多个本地预训练 ASR 模型：

| 模型 | 后端或脚本 | 用途 |
| --- | --- | --- |
| Qwen3-ASR-0.6B | `qwen-asr` 专用评测脚本 | 最终快模型层之一 |
| Qwen3-ASR-1.7B | `qwen-asr` 专用评测脚本 | 最终强模型回退层 |
| Paraformer-zh | FunASR | 最终快模型层之一，完整 test 对照模型 |
| SenseVoice-small | FunASR | 快模型消融对照 |
| Sherpa-ONNX Paraformer small | Sherpa-ONNX | CPU/ONNX 部署对照 |
| Vosk full | Vosk | 传统离线识别器对照 |
| Whisper tiny/base/small、Moonshine Tiny zh | Transformers 或可选后端 | 前期可运行性和弱基线探索 |

模型参数均来自本地预训练模型目录。工程改进来自推理阶段分级决策，没有重新训练或微调 ASR 模型参数。

## 5. 方法：多模型分级识别系统

最终系统结构如下：

```text
输入音频
  -> Qwen3-ASR-0.6B
  -> Paraformer-zh
  -> 中文文本规范化
  -> 若两个快模型规范化文本完全一致：采用 Qwen3-ASR-0.6B 输出
  -> 若两个快模型规范化文本存在分歧：调用 Qwen3-ASR-1.7B，并采用 1.7B 输出
```

该系统包含两层：

1. 快模型层：Qwen3-ASR-0.6B 与 Paraformer-zh。两者速度明显快于 Qwen3-ASR-1.7B，并且模型结构和后端来源不同，输出一致时通常表示样本较稳定。
2. 强模型回退层：Qwen3-ASR-1.7B。快模型分歧时，样本更可能存在发音、实体词、长句或局部歧义，系统调用强模型处理。

一致性门控的核心规则为：

```python
qwen06_norm = normalize_chinese_text(qwen06_text)
para_norm = normalize_chinese_text(paraformer_text)

if qwen06_norm == para_norm:
    final_text = qwen06_text
    qwen17_called = False
else:
    final_text = qwen17_text
    qwen17_called = True
```

选择 Qwen3-ASR-0.6B 作为快模型一致时的输出来源，主要为了保留其原始输出文本格式；在 CER 口径下，只要两个快模型规范化文本一致，采用任一快模型的规范化内容都会得到相同字符级结果。

平均耗时估计采用 batch summary 中的平均推理耗时，不计模型加载时间。最终系统在串行执行下的估计公式为：

```text
T = T_qwen0.6 + T_paraformer + p_call * T_qwen1.7
```

其中 `p_call` 为 Qwen3-ASR-1.7B 调用率。

## 6. 实验设计

实验分为三组。

第一组是完整 test 集上的可运行模型评测，用于建立统一基线与完整集结果。Qwen3-ASR-0.6B 和 Paraformer-zh 已在 full/test 上完成评测。

第二组是前期候选生成与选择实验。多视图音频实验对同一音频生成 `orig`、`peak_norm`、`trim_peak_norm`、`preemph`、`speed_0_95` 等视图，使用同一模型识别多次，再用 full/train 文本训练的字符 n-gram LM 重排序。该实验用于验证候选之间存在局部互补性。

第三组是 `representative100_eval/test` 上的强模型与多模型决策实验。由于 Qwen3-ASR-1.7B 单条平均耗时约 13 秒，强模型实验在 100 条受限评估子集上完成。该组实验比较单模型、最终分级系统、加入更多快模型的级联策略、加权投票，以及 oracle 理论上限。

oracle 仅作为理论上限分析，用来说明多模型候选之间存在互补性。最终系统没有使用 oracle，因为 oracle 在选择候选时依赖参考答案，实际推理阶段不可获得。

## 7. 实验结果

### 7.1 完整 test 集基线

完整 test 集结果来自指定 summary 文件：

| 方法 | 数据 | 样本数 | 总参考字符 | 编辑距离 | CER | 平均推理耗时 |
| --- | --- | ---: | ---: | ---: | ---: | ---: |
| Qwen3-ASR-0.6B | full/test | 2495 | 81139 | 3140 | 0.038699 | 1.9008s |
| Paraformer-zh | full/test | 2495 | 81139 | 3284 | 0.040474 | 0.1620s |
| Paraformer-zh 多视图重排序 | full/test | 2495 | 81139 | 3267 | 0.040264 | 见说明 |

数据来源：

| 方法 | summary 文件 |
| --- | --- |
| Qwen3-ASR-0.6B | `outputs/full/qwen_asr/qwen3-asr-0.6b/test_summary.json` |
| Paraformer-zh | `outputs/full/baseline/paraformer-zh/test_summary.json` |
| Paraformer-zh 多视图重排序 | `outputs/full/harness_v1/paraformer-zh/test_summary.margin015.json` |

多视图重排序将 Paraformer-zh 的 CER 从 0.040474 降至 0.040264，编辑距离从 3284 降至 3267。该提升较小，说明单模型多视图可以提供局部增益；该流程需要对同一音频运行多个视图，运行成本高于单次 Paraformer 推理，因此最终系统采用更直接的快模型一致性门控。

### 7.2 受限评估子集上的模型权衡

`representative100_eval/test` 含 100 条样本，总参考字符数为 3214。各模型结果如下。

| 模型 | 设备 | 编辑距离 | CER | 平均推理耗时 |
| --- | --- | ---: | ---: | ---: |
| Qwen3-ASR-1.7B | cuda:0 | 85 | 0.026447 | 13.0793s |
| Qwen3-ASR-0.6B | cuda:0 | 125 | 0.038892 | 1.6239s |
| Paraformer-zh | cuda:0 | 130 | 0.040448 | 0.1002s |
| SenseVoice-small | cuda:0 | 168 | 0.052271 | 0.0763s |
| Sherpa-ONNX Paraformer small | CPU | 231 | 0.071873 | 0.0976s |
| Vosk full | CPU | 233 | 0.072495 | 1.5404s |

结果显示，Qwen3-ASR-1.7B 的 CER 最低，但单条推理耗时明显高于快模型。Qwen3-ASR-0.6B 与 Paraformer-zh 的 CER 高于 1.7B，耗时低很多，适合作为分级系统的第一层。

### 7.3 最终分级系统

最终分级系统使用 Qwen3-ASR-0.6B 与 Paraformer-zh 做快模型一致性门控，分歧样本调用 Qwen3-ASR-1.7B。

| 方法 | CER | 编辑距离 | 1.7B 调用率 | 平均推理耗时估计 |
| --- | ---: | ---: | ---: | ---: |
| Qwen3-ASR-1.7B 单模型 | 0.026447 | 85 | 100% | 13.0793s |
| 多模型分级识别系统 | 0.026136 | 84 | 55% | 8.9177s |

最终系统的强模型调用统计来自 `outputs/representative100_eval/harness_final_eval/cascade_qwen06_para_exact/summary.json`：

| selected_source | 样本数 |
| --- | ---: |
| `qwen3-asr-1.7b` | 55 |
| `fast_exact:qwen3-asr-0.6b` | 45 |

耗时估计为：

```text
T = 1.6239 + 0.1002 + 0.55 * 13.0793
  = 8.9177s
```

相对每条都调用 Qwen3-ASR-1.7B：

```text
耗时降低比例 = (13.0793 - 8.9177) / 13.0793
           = 31.82%
```

CER 从 0.026447 降至 0.026136，编辑距离减少 1。该精度提升很小，主要工程价值来自在保持接近强模型 CER 的同时，将强模型调用率降至 55%。

### 7.4 消融与理论上限

在同一受限评估子集上，项目还比较了加入更多快模型的级联策略、加权投票和 oracle 理论上限。

| 方法 | 候选模型 | CER | 编辑距离 | 说明 |
| --- | --- | ---: | ---: | --- |
| `cascade_qwen06_para_exact` | 1.7B、0.6B、Paraformer | 0.026136 | 84 | 最终系统，1.7B 调用率 55% |
| `cascade_all_fast_exact` | 1.7B、0.6B、Paraformer、SenseVoice、Sherpa、Vosk | 0.026136 | 84 | 一致条件更严格，1.7B 调用率 87% |
| `weighted_all_models` | 全部上述模型 | 0.026758 | 86 | 加权字符投票出现误改 |
| `oracle_all_models` | 全部上述模型 | 0.018668 | 60 | 理论上限分析，实际系统未使用 |

`cascade_all_fast_exact` 与最终系统 CER 相同，但加入更多快模型后，所有快模型完全一致的概率下降，Qwen3-ASR-1.7B 调用率升至 87%。如果按串行推理估计，它还需要额外运行 SenseVoice、Sherpa-ONNX 和 Vosk，部署成本高于最终双快模型设计。

`weighted_all_models` 的 CER 为 0.026758，编辑距离为 86，低于强模型单独结果。其逐字投票会在局部位置引入误改，说明简单投票不适合作为最终策略。

`oracle_all_models` 的 CER 为 0.018668，编辑距离为 60。该结果说明不同模型候选之间存在明显互补性。例如 oracle 选择来源统计中，Qwen3-ASR-1.7B 贡献 79 条最优候选，Qwen3-ASR-0.6B 贡献 8 条，Paraformer-zh 贡献 8 条，Sherpa-ONNX 贡献 2 条，Vosk full 贡献 3 条。该结果只表示候选池理论空间，推理阶段没有参考答案，最终系统不使用 oracle。

### 7.5 典型样本分析

最终分级系统相对 Qwen3-ASR-1.7B 有 3 条规范化输出发生变化，其中 2 条改善，1 条变差。

| utt_id | 参考文本片段 | 1.7B 输出片段 | 分级系统输出片段 | 1.7B ED | 系统 ED | 结果 |
| --- | --- | --- | --- | ---: | ---: | --- |
| D32_764 | 乔与她山西相恋共同返城 | 巧与他山西相恋共同返程 | 乔与他山西相恋共同返程 | 3 | 2 | 改善 |
| D6_788 | 长寿得利于您特别健康 | 长寿得力于您特别健康 | 长寿得利于您特别健康 | 1 | 0 | 改善 |
| D6_978 | 亲王僧格林沁 | 亲王僧格林沁 | 亲王森格林沁 | 0 | 1 | 变差 |

这些样本体现了一致性门控的收益和风险。快模型一致时可以修正强模型的个别局部错误，例如“得力于/得利于”；快模型也可能在实体词上共同出错，例如“僧格林沁/森格林沁”。最终系统采用完全一致门控，已经较为保守，但仍存在少量误选风险。

## 8. 分析

第一，模型规模与运行成本之间存在明显权衡。Qwen3-ASR-1.7B 在受限评估子集上的 CER 最低，为 0.026447，平均推理耗时 13.0793s。Qwen3-ASR-0.6B 和 Paraformer-zh 的 CER 分别为 0.038892 与 0.040448，耗时分别为 1.6239s 与 0.1002s。单独使用快模型会牺牲精度，单独使用强模型会增加平均运行时间。

第二，双快模型一致性是有效的门控信号。最终系统中 45% 样本由快模型一致性直接处理，55% 样本回退强模型。该策略将 Qwen3-ASR-1.7B 调用率从 100% 降至 55%，同时 CER 与强模型单独运行非常接近，并在该子集上减少 1 个编辑距离。

第三，增加模型数量需要控制门控条件和推理成本。`cascade_all_fast_exact` 在 CER 上与最终系统相同，但强模型调用率升至 87%，并额外运行多个模型。全模型加权投票引入误改，说明候选越多不必然带来更好的自动选择结果。

第四，oracle 结果表明候选互补性仍有空间。全模型 oracle 的 CER 为 0.018668，显著低于 Qwen3-ASR-1.7B 的 0.026447。该差距说明如果未来能设计更可靠的无参考候选选择器，多模型系统仍可能进一步降低 CER。

第五，当前结论需要控制外推范围。`representative100_eval/test` 是高耗时强模型实验的受限评估子集，样本数为 100。最终分级系统的 CER 改善只有 1 个编辑距离，统计强度有限；完整 test 集上的强模型结果仍需在资源允许时补充。

## 9. 复现路径

环境使用 Python 3.11，依赖记录见 `harness/requirements.txt`。CUDA 版 PyTorch、FunASR、Transformers、qwen-asr、Sherpa-ONNX、Vosk 等依赖均通过本地虚拟环境运行。

数据准备与校验：

```powershell
.\.venv\Scripts\python.exe harness\scripts\prepare_manifests.py
.\.venv\Scripts\python.exe harness\scripts\validate_manifests.py
```

完整 test 集基线：

```powershell
.\.venv\Scripts\python.exe harness\scripts\evaluate_qwen_asr.py --model-path models/qwen3-asr-0.6b --model-name qwen3-asr-0.6b --manifest data/manifests/full/test.jsonl --device cuda:0

.\.venv\Scripts\python.exe harness\scripts\evaluate_baseline.py --model paraformer-zh --manifest data/manifests/full/test.jsonl --device cuda:0
```

受限评估子集模型评测：

```powershell
.\.venv\Scripts\python.exe harness\scripts\evaluate_qwen_asr.py --model-path models/qwen3-asr-1.7b --model-name qwen3-asr-1.7b --manifest data/manifests/representative100_eval/test.jsonl --device cuda:0

.\.venv\Scripts\python.exe harness\scripts\evaluate_qwen_asr.py --model-path models/qwen3-asr-0.6b --model-name qwen3-asr-0.6b --manifest data/manifests/representative100_eval/test.jsonl --device cuda:0
```

最终分级系统离线评测：

```powershell
.\.venv\Scripts\python.exe harness\scripts\evaluate_ensemble_harness.py `
  --manifest data\manifests\representative100_eval\test.jsonl `
  --prediction qwen3-asr-1.7b=3.0=outputs\representative100_eval\qwen_asr\qwen3-asr-1.7b\test_predictions.jsonl `
  --prediction qwen3-asr-0.6b=2.0=outputs\representative100_eval\qwen_asr\qwen3-asr-0.6b\test_predictions.jsonl `
  --prediction paraformer-zh=1.0=outputs\representative100_eval\baseline\paraformer-zh\test_predictions.jsonl `
  --anchor qwen3-asr-1.7b `
  --strategy cascade_fast_exact `
  --output-root outputs `
  --harness-name harness_final_eval `
  --output-name cascade_qwen06_para_exact
```

## 10. 结论

本项目完成了一个中文自动语音识别多模型分级识别系统。工程统一了 manifest、模型后端、预测 JSONL、summary JSON、中文文本规范化、CER 计算、批量评测入口和单条音频识别入口。最终系统使用 Qwen3-ASR-0.6B 与 Paraformer-zh 构成快模型层，使用 Qwen3-ASR-1.7B 构成强模型回退层，通过快模型规范化文本完全一致这一信号控制强模型调用率。

在 `representative100_eval/test` 受限评估子集上，最终系统将 Qwen3-ASR-1.7B 调用率降至 55%，串行估计平均推理耗时从 13.0793s 降至 8.9177s，CER 从 0.026447 降至 0.026136。该精度提升幅度很小，但在降低平均运行时间和强模型调用率的同时保持了接近强模型的低 CER。

当前系统的主要局限是强模型完整 test 集结果尚未补齐，最终分级系统只在 100 条受限评估子集上验证。后续工作可以在更大评估集上补充 1.7B 结果，并研究更可靠的无参考候选选择器，以利用 oracle 分析中显示的多模型互补空间。
